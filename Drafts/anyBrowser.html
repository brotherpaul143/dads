Viewable with any Browser

DADS bears the "Viewable with any Browser" icon because the authors of DADS agree with this campaign. We are participating in this effort by avoiding tricks that may influence the transportability of this website. We are aware that it may look better in some browsers than others, but are confident they are readable by the vast majority of browsers in use.

The following highlits some unavoidable problems and how we address them:

  • Fonts are not standardised. Most browsers are dependant on fonts that are globally available on the machine where the browser is running. Furthermore, different browsers have different heuristics to determine the fall-back font to use if a specified font is not available. We aim to achieve maximum consistency accross browsers, we do not make extensive use of fonts. When we specify fonts, we use only fonts that are widely used and hence more likely to be available. We explicitly specify a fall-back chain of fonts down to the best known generic font in the font-family we prefer.
  • Authors of websites as well as readers can specify style preferences. A user's style prefrences my override style specifications for our site and style preferences that were specified for previously viewed sites may have an effect on how our site is rendered. We aim to minimise the possibility that these side effects have a negative impact on the viewability of our site by minimizing the number of styles we rely on and by resetting styles that we are aware of that may have a harmful impact on our site.

If you experience any difficulty in viewing any information in DADS in your preferred way, we would like to know about it. include(`Pages/bitsFixDefn.m4')

The following is a list of other things we do to support the viewability of this site:

  • Formatting specific settings are set in a cascading style sheet. If a browser does not support cascading style sheets, the page will be rendered with default settings for all tags, whis is in most cases acceptable.
  • Formatting specific settings are kept to a minimum and only used when we have conformation that it is supported by most browsers. A good reference we use for this purpose is at: http://reference.sitepoint.com/css
  • We minimise the use images. When we use images, the dimensions of the image is specified in pixels, an alternate text describing the images is included that can be displayed if for some reason the image is not shown. We use only JPEG format because it seems to be the most widely supported image format that is not limited by copyright issues.
  • We consistently use only lower case in HTML tags. Although, HTML is not case sensitive, XHTML requires all tags to be in lowercase. This helps to have more portable code.
  • Reserved characters in URL's specified in outgoing links are encoded using their hexadecimal format.
